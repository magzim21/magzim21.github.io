TOP command

This is command for live process monitoring
Shortcuts:
k kill process
M to sort by memory usage
m change memory info view
P to sort by CPU usage
N to sort by process ID
T to sort by the running time
t change cpu usage view
H switch from process to threads counting
E switch memoty counting. eg bytes to gb
c show full path of the process' executable. Kernelspace processes are marked with square brackets around them.
V forest view to see the process' hierarchy.
W to save the current configuration to ~/.toprc, so next time we will have same view
z show colorized view
u processes of a certain user
Interpreting . I suggest to refet to man pages.
$CPU is usage of one cpu. It can be higher than 100%. This is evalueted for one core. And this is accurate metric btw. A system with 1core and a load average of 2.12 means that it is overloaded by 112% more work than it can handle.
In order to change evaluation to 100% is num of cpu press shift + i - avarage metrics will not change instantly, but evaluation will do.
memory total = free + used + buff/cache
buff/cache is a memory occupied by most frequently used files. It is considered as free. As soon as process needs more space, temp files stored here will be deleted.

CPU metrics

24.8 us time running user space processes spent.
0.5 sy - This is the amount of time that the CPU spen trunning the kernel. This number can peak much higher, especially when there is a lot of I/O happening
0.0 ni how much time the CPU spent running user space processes that have been niced
73.6 id processor was idle just over 73% of the time during the last sampling period
0.4 wa time cpu was idle while waiting for an I/O operation to complete. CPU is much faster then io.
0.0 hi & 0.2 si how much time the processor has spent servicing interrupts. hi is for hardware interrupts, and si is for software interrupts
0.0 st show long the virtual CPU has spent waiting for the hypervisor to service another virtual CPU

Free command

It does not show more than 'top' command
By default it's calculated based on power of 1024 (not 1,000). EG it will show mebibytes instead of mgabytes
"buff/cache" is memory that Linux uses for disk caching, and that will be freed when applications require it. So you don't have to worry if a large amount is being shown in this field.
use man page to see explanation of each column
Available column = how much memory is available for starting new applications, without swapping.
Used colum = total - free - buff/cache
t : display total of columns (swap+Mem)
